How it works

From zero to running test in under two minutes.

1

Prove you own the target

Upload a small verification file to your server (same idea as Let's Encrypt domain validation). This stops anyone from using LoadTester as a DDoS weapon against targets they don't own.

2

Send a POST request

Call POST /v1/tests with your target URL, number of virtual users, duration, and the locations you want traffic from. That's it.

3

Workers spin up worldwide

The master assigns the test to workers in your chosen locations. Each worker ramps up VUs gradually to simulate real traffic patterns.

4

Stream live metrics, then get the report

Workers report RPS, latency percentiles, and error counts every second. When the test finishes, GET /v1/tests/:id/report returns the full structured report.

Ownership verification

We verify you control the target domain before sending any load. This is non-negotiable — it protects the internet from abuse.

How to verify your domain

Create a static text file at a well-known URL on your server. We fetch it before the test starts. If it returns HTTP 200, you're verified.

  1. Call POST /v1/prepare-verify with your target_url to get a unique token, e.g. lt-9f3c2a1b
  2. Create the file /.well-known/verify-loadtest-lt-9f3c2a1b.txt on your server — content can be anything, just needs to return 200.
  3. Pass verification_file_url in your POST /v1/tests request. We fetch it, confirm 200, then start the test.

Quick start

Two API calls. No SDK required.

Run a test with curl

After placing your verification file, start a test:

  1. curl -X POST https://api.loadtester.io/v1/tests \
      -H "Content-Type: application/json" \
      -d '{"target_url":"https://api.example.com","virtual_users":500,"duration_seconds":60,"verification_file_url":"https://api.example.com/.well-known/verify-loadtest-YOUR-TOKEN.txt"}'
  2. Poll for results:
    curl https://api.loadtester.io/v1/tests/YOUR-TEST-ID
  3. Get the full report:
    curl https://api.loadtester.io/v1/tests/YOUR-TEST-ID/report
