Anti-DDoS protection

Domain verification

Before any test begins, you must prove you control the target domain. This prevents LoadTester from being used as a DDoS tool against sites you don't own.

1

Start a test — get your verification ID

When you configure a test, we generate a unique test ID (e.g. lt_a3f9b2c1).

2

Serve a file at the verification path

Create a file at /.well-known/loadtester-verify/{test-id} with the content we show you.

# Example: serve it with any web server mkdir -p /var/www/.well-known/loadtester-verify echo "lt_a3f9b2c1:verified:2026" > /var/www/.well-known/loadtester-verify/lt_a3f9b2c1 # Or with nginx, just add: location /.well-known/loadtester-verify/ { root /var/www; }
3

Click "Verify" — we check it and proceed

We fetch the file to confirm it's there. Once verified, you can pay and launch the test. The file can be removed after the test ends.
